# 計画〜実装フロー

標準のPlanModeのプロセスに追加で以下を行う。

## 計画時

1. `/docs` および `/packages/*/docs/` の関連ドキュメントを参照
2. 計画を立てたら `/plans` に保存
   - ファイル名: `yyyy-mm-dd-hh-mm_プラン名.md`
   - 例: `2026-01-30-14-30_add-gradient-support.md`
3. 計画を更新したらmdもアップデート

## 計画更新・軌道修正・追加対応時

- 実装中に方針変更や追加の観点が出た場合はmdに反映
- 変更理由も簡潔に記録

## 完了時

ユーザーが実装結果を確認した上でコミットを指示したら：

1. mdに実装結果を簡潔にまとめる
2. ペンディング事項があれば明記
3. 影響範囲のドキュメント更新漏れがないことを確認
4. 簡潔な日本語で全てをコミット
